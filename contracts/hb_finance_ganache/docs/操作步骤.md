# 本地 Ganache 与 Sepolia 操作步骤

## 一、本地 Ganache 部署

```powershell
cd /d D:\hb_finance_ganache
npm.cmd run chain
```

新开一个 PowerShell：

```powershell
cd /d D:\hb_finance_ganache
npm.cmd run deploy:local
npm.cmd run switch:local
```

部署结果在：

```text
D:\hb_finance_ganache\deployments\local-latest.json
D:\hb_finance_ganache\env\frontend.local.env
```

前端环境写入：

```text
C:\Users\Administrator\heibai-workspace\projects\hb-chain-finance\.env.local
```

## 二、MetaMask 添加本地网络

1. 打开 MetaMask。
2. 网络选择器选择“添加自定义网络”。
3. 网络名称：`HB Finance Local Ganache`
4. RPC URL：`http://127.0.0.1:8545`
5. Chain ID：`31337`
6. 货币符号：`ETH`
7. 保存并切换到该网络。

## 三、导入 Ganache 测试账户

MetaMask 选择“导入账户”，使用本地 Ganache 启动日志中输出的测试账户私钥。

注意：
- 不要把真实钱包私钥、助记词、后台密码或 RPC 密钥写入仓库。
- Ganache 测试账户只用于本地私有链，不得用于真实资产。

## 四、前端本地调试

```powershell
cd /d C:\Users\Administrator\heibai-workspace\projects\hb-chain-finance
npm.cmd run dev
```

打开前端后进入储蓄/Deposit 页面，连接 MetaMask。本地模式会要求切换到 Chain ID `31337`，资产地址和 TestAssetManager 地址来自 `.env.local`。

USDT 授权流程固定为：

```text
approve(spender, 0)
approve(spender, amount)
```

USDC 和 pYUSD 授权流程为：

```text
approve(spender, amount)
```

## 五、Sepolia 演示切换

```powershell
cd /d D:\hb_finance_ganache
npm.cmd run switch:sepolia
```

然后在前端项目构建：

```powershell
cd /d C:\Users\Administrator\heibai-workspace\projects\hb-chain-finance
npm.cmd run build
```

Vercel 演示使用 Sepolia 环境变量：

```text
NEXT_PUBLIC_WEB3_ENV=sepolia
NEXT_PUBLIC_EVM_CHAIN_ID=11155111
NEXT_PUBLIC_USDT_ADDRESS=0xd78e26e0017fb6D830395f5247c04CC71DEc3a47
NEXT_PUBLIC_USDC_ADDRESS=0x4229D50d940E546DEaa8fc1fEe5b72cCCE3E9DbF
NEXT_PUBLIC_PYUSD_ADDRESS=0x434aA6660FA949A9d57128beCe17DD33E4899b27
NEXT_PUBLIC_ASSET_MANAGER_ADDRESS=0x57C3a549e3aFa9c12fE9031F1ADE08A8D8729A28
NEXT_PUBLIC_USDT_REQUIRES_ZERO_APPROVAL=true
```

本地模式用于无限币调 bug；Sepolia 模式用于打包部署 Vercel 给项目方演示。
